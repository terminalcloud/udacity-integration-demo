Terminal Integration Demo

The purpose of this demo is to illustrate that our code allows us to inject a configurable collaborative web IDE into any webpage.

Behind the scenes, the system is:

  • using Udacity's JWT to authenticate with our servers
  • allocating an isolated hosted environment in the cloud
  • connecting to that environment, and cloud storage, to continuously save files and execute code on demand by the user

On the front end:

We threw together this demo to show how quickly and easily Terminal's IDE could be integrated into Udacity's coursework. In production, Udacity would embed Terminals into the page using HTML snippets your CD's would publish easily with your regular publishing tools e.g. Coco. For demonstration purposes, we've provided you with a bookmarklet so you can see live Terminals embedded in your page right now.

Technical details: The Bookmarklet injects our javascript library into the page, and a div into the DOM at a location specific to Udacity's layout. Subsequently, a React component is rendered into that div. The component is itself made up of several smaller modular components such as "files browser", "file editor", "terminal", "preview pane", and so on, which can themselves be used independently of one another. In addition, the components can be rearranged and separated as best fits your instructional layouts, and can communicate with each other even if physically separated on the page. For instance, you could include a button that changes the source code file the student is looking at in the editor, or unfolds some code.

At the top level, there is a flexible layout engine that allows resizing of the various panels.

The component panels don't have to be next to each other or even laid out by our layout engine, that's just what's happening in this one example. You can also have several instances of any particular panel and you can script your own complex interactions between panels and between your own code and the panels.
