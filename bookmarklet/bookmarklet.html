Terminal Integration Demo

The purpose of this demo is to illustrate that our code allows us to inject a configurable collaborative web IDE into any webpage.

Behind the scenes, the system is:

  • using Udacity's JWT to authenticate with our servers
  • allocating an isolated hosted environment in the cloud
  • connecting to that environment, and cloud storage, to continuously save files and execute code on demand by the user

On the front end:

The Bookmarklet injects our javascript library into the page, and a div into the DOM at a location specific to Udacity's layout. Subsquently, a React component is rendered into that div. The component is itself made up of several smaller modular components such as "files browser", "file editor", "terminal", "preview pane", and so on, which can themselves be used independently of one another. At the top level, there is a flexible layout engine that allows resizing of the various panels.

The component panels don't have to be next to each other or even laid out by our layout engine, that's just what's happening in this one example. You can also have several instances of any particular panel and you can script your own complex interactions between panels and between your own code and the panels.

We threw together this demo to show how quickly and easily Terminal's IDE could be integrated into Udacity's coursework.
